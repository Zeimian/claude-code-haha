# 毛泽东思想 × Superpowers 工作流集成指南

## 综合理念
将毛泽东思想的实践哲学与 Claude Code 中实际存在的 Superpowers 风格架构相结合，形成高效、实用、可验证的开发范式。

基于对 Claude Code 代码库的实际分析，Superpowers 的核心理念已在以下组件中体现：协调器模式、代理委派系统和技能工具。

---

## 一、实事求是 + 协调器模式

### 核心原则
- **深入调查**: 在任何开发开始前，通过协调器进行全面分析
- **基于事实**: 不凭主观想象开发，必须基于真实项目状态

### 实践方法
1. **协调器分析**
   - 在复杂任务开始时，系统自动启动协调器模式
   - 协调器会分析任务的各个方面

2. **并行研究**
   - 启动多个代理并行调查不同方面
   - 遵循 "没有调查就没有发言权"

3. **综合验证**
   - 所有假设必须基于多代理的研究结果
   - 不做未经验证的设计决策

---

## 二、矛盾论 + 代理委派

### 核心原则
- **识别主要矛盾**: 通过并行代理找出制约项目的瓶颈
- **集中资源**: 优先解决核心问题

### 实践方法
1. **多代理研究**
   - 并行识别不同类型的矛盾
   - 技术债务 vs 功能开发
   - 性能瓶颈 vs 新功能

2. **优先级排序**
   - 重要且紧急：立即处理
   - 重要不紧急：计划处理
   - 不重要但紧急：委托或延后

3. **资源集中分配**
   - 重点解决主要矛盾
   - 暂时搁置次要问题

---

## 三、实践论 + 4阶段管道

### 核心原则
- **小步快跑**: 通过分阶段管道逐步验证
- **边做边验证**: 每个阶段都进行验证

### 实践方法
1. **4-阶段执行**
   - 研究 → 综合 → 实施 → 验证
   - 每个阶段完成后进入下一阶段

2. **迭代验证**
   - 研究阶段验证可行性
   - 实施阶段验证正确性
   - 验证阶段验证完整性

3. **反馈循环**
   - 每个阶段收集反馈
   - 根据反馈调整后续阶段

---

## 四、群众路线 + 技能系统

### 核心原则
- **代码为民服务**: 代码要便于他人阅读和维护
- **从群众中来**: 遵循团队编码规范
- **到群众中去**: 确保代码符合团队需求

### 实践方法
1. **技能重用**
   - 使用团队验证过的技能（如 /commit, /review）
   - 遵循团队最佳实践

2. **协作精神**
   - 多代理协作完成任务
   - 互相验证结果

3. **知识传递**
   - 好的实践通过技能系统共享
   - 技术决策透明化

---

## 五、战略战术 + 代理架构

### 核心原则
- **战略上藐视**: 任何复杂问题都可以通过多代理解决
- **战术上重视**: 每个代理的任务都要认真对待

### 实践方法
1. **战略规划**
   - 有信心解决复杂问题
   - 将大问题分解为多代理任务

2. **战术执行**
   - 每个代理的任务都要认真完成
   - 详细的验证和测试

3. **灵活应变**
   - 代理任务可调整
   - 可随时改变策略

---

## 六、反本本主义 + 实际执行

### 核心原则
- **反对过度设计**: 遵活使用内置功能
- **反对形式主义**: 关注实际效果而非表面工程

### 实践方法
1. **实用导向**
   - 功能能用就行，不要过度工程
   - 简洁优于复杂

2. **适度委派**
   - 简单任务无需多代理
   - 复杂任务才启动协调器模式

3. **拒绝炫技**
   - 技术选择以解决实际问题为准
   - 不盲目使用高级功能

---

## 七、游击战术 + 代理调度

### 核心原则
- **灵活机动**: 适应需求变化
- **集中优势兵力**: 通过多代理并行
- **积小胜为大胜**: 通过分阶段完成

### 实践方法
1. **小步快跑**
   - 每个代理任务具体明确
   - 快速验证每个阶段
   - 及时调整方向

2. **适应变化**
   - 需求变化时调整代理任务
   - 优先级动态调整

3. **持续改进**
   - 每次完成都总结经验
   - 不断优化代理协作

---

## 八、三大纪律 + 系统使用

### 纪律一：一切行动听指挥
- **严格执行需求**: 遵循用户的真实需求，理解偏差为零
- **不偏离目标**: 始终围绕核心目标工作

### 纪律二：不拿群众一针一线
- **不随意修改**: 只修改任务相关的代码
- **不引入无关依赖**: 保持项目清洁

### 纪律三：一切缴获要归公
- **分享发现**: 将有用的工具、方法记录下来
- **贡献社区**: 好的实践推广给团队

---

## 九、综合工作流程（基于 Claude Code 实际实现）

1. **任务识别阶段 (协调器启动)**
   - 实事求是：深入调研任务需求
   - 群众路线：理解真实用户需求

2. **代理委派阶段 (Agent Tool)**
   - 矛盾论：识别主要矛盾
   - 战略战术：制定执行计划

3. **执行阶段 (多代理并行)**
   - 实践论：小步快跑，边做边验证
   - 游击战术：灵活适应变化

4. **验证阶段 (结果验证)**
   - 群众路线：团队协作保证质量
   - 反本本主义：关注实际效果

5. **总结阶段 (结果汇报)**
   - 实践论：总结经验教训
   - 群众路线：分享知识

## 十、效果检验标准

使用这套综合方法后，应该能看到：

1. **减少过度设计**: 不再为简单功能启动复杂代理
2. **提高开发效率**: 通过多代理并行提高效率
3. **改善协作**: 统一的工作标准提高协作效率
4. **提升质量**: 通过分阶段验证提高代码质量
5. **增强实用**: 更贴近真实需求，解决实际问题
