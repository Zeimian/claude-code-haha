# Superpowers 工作流 - 快速参考指南

## 概述
Superpowers 是一个结构化的工作流程系统，由 14 个核心技能组成，旨在提高代码质量和开发效率。

## 14 个核心技能详解

### 1. 需求分析 (Brainstorming)
- **目的**: 深入理解用户需求
- **方式**: 苏格拉底式提问（每次只问一个问题）
- **输出**: 详细的需求规格说明

**毛泽东思想应用**:
- 实事求是：基于真实需求，不凭空想象
- 群众路线：理解用户的真实需求

### 2. 工作区隔离 (Using Git Worktrees)
- **目的**: 创建安全的开发环境
- **方式**: 从主分支创建新分支进行开发
- **输出**: 隔离的工作空间

**毛泽东思想应用**:
- 集中优势兵力：在隔离环境中专注解决问题

### 3. 任务分解 (Writing Plans)
- **目的**: 将复杂任务分解为小步骤
- **方式**: 2-5分钟可完成的小任务
- **输出**: 详细任务清单

**毛泽东思想应用**:
- 矛盾论：识别并优先处理主要矛盾
- 实践论：从小任务开始验证

### 4. 子代理开发 (Subagent Driven Development)
- **目的**: 分工协作完成复杂任务
- **方式**: 启动多个子代理执行不同任务
- **输出**: 协作完成的代码

**毛泽东思想应用**:
- 集中优势兵力：分工协作完成复杂任务

### 5. 代码审查 (Requesting Code Review)
- **目的**: 保证代码质量
- **方式**: 自动和人工审查相结合
- **输出**: 审查报告和改进建议

**毛泽东思想应用**:
- 实践论：通过审查验证代码质量

## 工作流程实践

### 启动 Superpowers 工作流
当开始一个新项目时，可以按以下步骤启动 Superpowers 工作流：

1. **需求分析阶段**
   - 使用"Brainstorming"技能深入理解需求
   - 遵循实事求是原则

2. **设计规划阶段**
   - 使用"Writing Plans"技能分解任务
   - 应用矛盾论识别主要问题

3. **开发阶段**
   - 使用"Subagent Driven Development"进行协作开发
   - 保持实践论的态度，边开发边验证

4. **审查阶段**
   - 使用"Code Review"技能确保质量
   - 体现群众路线，让团队成员参与

## 实际应用建议

### 何时使用
- 新项目启动
- 复杂功能开发
- 重构项目
- 团队协作开发

### 使用技巧
- 在每个阶段结束时，确认目标达成后再进入下一阶段
- 不要跳过任何一个阶段，即使是看似简单的任务
- 每个阶段的输出都是下一阶段的输入

## 与毛泽东思想结合要点

### 实事求是
- 深入了解项目现状
- 基于实际情况制定计划
- 不凭主观愿望开发

### 矛盾论
- 识别主要技术难点
- 集中资源解决关键问题
- 优先处理主要矛盾

### 实践论
- 小步快跑，逐步验证
- 边开发边测试
- 从实践中获得反馈

### 群众路线
- 代码审查考虑团队其他成员
- 遵循团队编码规范
- 便于后续维护

## 工作纪律
- 遵守三大纪律：理解需求零偏差、不修改无关代码、分享最佳实践
- 反对本本主义：不盲目追求复杂架构
- 保持谦虚态度：接受审查意见，持续改进
