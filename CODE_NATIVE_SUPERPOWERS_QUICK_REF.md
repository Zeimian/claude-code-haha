# Claude Code Superpowers 工作流 - 基于实际实现的快速参考

## 概述
根据对 Claude Code 代码库的分析，Superpowers 风格的工作流已在其架构中体现。以下是基于实际实现的功能快速参考。

## 核心组件

### 1. 协调器模式 (Coordinator Mode)
- **目的**: 中央协调多个工作者代理
- **实现方式**: 自动启动，无需特殊命令
- **输出**: 合成多代理的结果

### 2. 代理委派 (Agent Tool)
- **目的**: 启动多个并行子代理
- **实现方式**: 使用 Agent() 函数
- **输出**: 分布式任务执行

### 3. 技能系统 (Skill Tool)
- **目的**: 调用可重用功能
- **实现方式**: /command 方式
- **输出**: 预定义功能执行

## 实际工作流程

### 4-阶段管道
1. **研究 (Research)**
   - 多个工作者并行调查
   - 利用并行主义提高效率

2. **综合 (Synthesis)**
   - 协调器分析多代理结果
   - 创建实施规范

3. **实施 (Implementation)**
   - 工作者根据规范执行
   - 分工协作完成任务

4. **验证 (Verification)**
   - 验证实施效果
   - 确认目标达成

## 实际应用建议

### 何时使用
- 复杂项目启动
- 需要并行研究的任务
- 需要多方协作的开发

### 使用技巧
- 让系统自动委派代理，无需手动干预
- 在任务中明确表达需要并行处理的要求
- 利用内置的技能命令（如 /commit, /review）

## 与毛泽东思想结合

### 实际体现
- **并行主义**: 体现了集中优势兵力的战略思想
- **分工协作**: 体现群众路线，集体智慧
- **分阶段验证**: 体现了实践论，边做边验证

### 工作纪律
- **三大纪律**:
  1. 理解需求零偏差
  2. 不修改无关代码
  3. 分享最佳实践

## 使用示例

当您在 Claude Code 中处理复杂任务时，系统会自动：
1. 识别任务的复杂性
2. 启动协调器模式
3. 委派多个代理并行处理
4. 综合结果并汇报
5. 逐步验证完成

这种方式自然地将 Superpowers 的理念融入到日常使用中，无需特殊配置。
