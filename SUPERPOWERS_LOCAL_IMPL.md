# Superpowers 技能系统 - 本地实现

## 系统概述
这是一个本地化的 Superpowers 技能系统，基于对 Claude Code 代码库的分析，为 Claude Code 提供结构化的工作流程。实际的 Superpowers 核心概念在 Claude Code 的编排系统中已有体现。

## 核心架构组件（基于 Claude Code 实际实现）

### 1. 协调器 (Coordinator)
- **用途**: 作为中央协调员管理多个工作者
- **方法**: 通过 Agent 工具生成工作者来处理并行任务
- **输出**: 合成结果并与用户沟通

### 2. 代理工具 (Agent Tool)
- **用途**: 核心委派机制，生成专门的子代理执行任务
- **方法**: 
  - 工作者异步运行并可执行研究、实施或验证任务
  - 支持不同的代理类型和专门功能
  - 启用独立任务的并行执行
- **输出**: 分工协作完成开发任务

### 3. 技能工具 (Skill Tool)
- **用途**: 提供调用可重用技能/命令的统一接口
- **方法**: 
  - 内置技能（如 `/commit`, `/review`, `/ship`）
  - 用户定义的自定义技能
  - 外部服务器的 MCP 技能
- **输出**: 可重用的功能模块

## 实际工作流程（基于 Claude Code 分析）

### 4-阶段执行管道
1. **研究阶段** - 多个工作者并行调查问题的不同方面
2. **综合阶段** - 协调员分析结果并创建具体的实施规范
3. **实施阶段** - 工作者根据综合规范执行目标更改
4. **验证阶段** - 工作者验证更改是否按预期工作

## 基于 Claude Code 的工作流示例

```
用户需求 → 协调器分析 → 并行委派多个代理 → 研究/实施/验证 → 结果综合 → 用户交付
```

每个阶段都利用了 Claude Code 的内置功能，如 Agent 工具实现并行处理，Skill 工具提供可重用功能。

## 与毛泽东思想的结合

- **并行主义**: 体现了集中优势兵力的战略思想
- **分工协作**: 体现群众路线，集体智慧
- **分阶段验证**: 体现了实践论，边做边验证
- **小步快跑**: 体现游击战术的灵活应变

## 实际应用方式

在 Claude Code 中，您可以：

1. 当您开始一个复杂任务时，系统会自动调用协调模式
2. 系统将自动委派多个代理进行并行处理
3. 研究、实施、验证等多个阶段会自动流转
4. 每个阶段完成后自动进入下一阶段
